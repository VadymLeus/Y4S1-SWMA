left to right direction

' --- Актори ---
' Визначаємо акторів системи
actor "Гість" as Guest
actor "Користувач" as User
actor "Адміністратор" as Admin

' --- Ієрархія акторів ---
' Користувач успадковує всі можливості Гостя
User --|> Guest
' Адміністратор успадковує всі можливості Користувача
Admin --|> User

' --- Варіанти використання ---

' Загальні дії
Guest -- (Перегляд каталогу сайтів)
(Перегляд каталогу сайтів) ..> (Пошук сайту) : <<extend>>
Guest -- (Перегляд публічного сайту)
Guest -- (Реєстрація)
Guest -- (Авторизація)

' Дії Користувача
User -- (Керування профілем)
User -- (Додавання сайту в 'Обране')
User -- (Створення тікету підтримки)
' Розширення: Оскарження - це особливий випадок створення тікету
(Оскарження блокування) .> (Створення тікету підтримки) : <<extend>>
User -- (Оскарження блокування)

' Керування Власним Сайтом
User -- (Керування сайтами)
' Узагальнення: Керування сайтом складається з...
(Керування сайтами) <|-- (Створення сайту)
(Керування сайтами) <|-- (Редагування контенту)
(Керування сайтами) <|-- (Редагування налаштувань)
(Керування сайтами) <|-- (Видалення сайту)
' Включення: Створення сайту неможливе без вибору шаблону
(Створення сайту) ..> (Вибір шаблону) : <<include>>
' Розширення: Редагування контенту може включати...
(Редагування контенту) ..> (Керування товарами) : <<extend>>
(Редагування контенту) ..> (Керування категоріями) : <<extend>>

' Функції Магазину (Клієнт)
User -- (Додавання товару в кошик)
User -- (Керування кошиком)
User -- (Оформлення замовлення)

' Адміністрування
Admin -- (Обробка тікетів підтримки)
Admin -- (Модерація сайтів)
Admin -- (Видача попереджень користувачам)
Admin -- (Перегляд повного списку сайтів)

@enduml
